# Agent Definition Of Done

- Purpose: Define the minimum completion bar for any feature delivered against this documentation package.
- Audience: Coding agents, engineers, and reviewers.
- Authority: Canonical agent-enablement document.
- Owner: Project maintainers and implementation agents.
- In scope: Engineering done criteria.
- Out of scope: Business sign-off.
- Read with: `.ai/31-acceptance-criteria.md`, `.ai/35-production-readiness-checklist.md`.

## Done Criteria

- `Confirmed`: Behavior matches the canonical `.ai/*` docs.
- `Confirmed`: Contracts are explicit and runtime-validated.
- `Confirmed`: Tests cover success, validation failure, dependency failure, and cache or refresh behavior.
- `Confirmed`: Logs and metrics cover the new path.
- `Confirmed`: Docs stay truthful after the change.
- `Confirmed`: No dependency or pattern from `.ai/initial/*` is revived without a new recorded decision.
